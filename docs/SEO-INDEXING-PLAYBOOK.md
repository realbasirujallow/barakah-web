# SEO Indexing Playbook

_Round 33 SEO submission pass. This is the canonical reference for how the
Barakah web app gets into Google/Bing/Yandex/Seznam/Naver._

---

## TL;DR

1. **Sitemap** lives at `https://trybarakah.com/sitemap.xml` — 55 URLs, auto-generated by Next.js.
2. **robots.txt** at `https://trybarakah.com/robots.txt` references the sitemap, allows search crawlers, blocks AI training bots.
3. **IndexNow** is wired up via `.github/workflows/indexnow.yml` — fires automatically on every push to `main` that touches a page / layout / sitemap. Submits to Bing, Yandex, Seznam.
4. **Google** does NOT support IndexNow. Follow the one-time steps in [§ 3](#3-google-search-console-one-time-manual) below to submit the sitemap via Search Console.

## 2. IndexNow — automatic (Bing, Yandex, Seznam, Naver, Mojeek)

### How it works

1. Your repo owns a key file: `public/140db6db4ee38df9d39565683ff0b1734617c449d136e11d8e6fb0d7502663ab.txt`.
2. The same key is embedded in `.github/workflows/indexnow.yml` as the `key` field of the JSON payload.
3. Every push to `main` that changes `src/app/**/page.tsx`, `src/app/**/layout.tsx`, or the sitemap files triggers the workflow.
4. The workflow waits 90 s for the Vercel deploy, then POSTs either (a) the touched URLs or (b) ALL sitemap URLs (`workflow_dispatch` with `full_resubmit=true`) to the three IndexNow endpoints.
5. The IndexNow provider fetches the key file on your domain to verify ownership → HTTP 200 → queued for crawling.

### Key rotation

**Don't rotate without a reason.** The key is tied to the domain forever for IndexNow purposes. If it leaks (it's public-by-design anyway), you simply stop being able to submit as `trybarakah.com` — attackers can't harm indexing.

If you must rotate:
1. Generate a new key: `openssl rand -hex 32`.
2. Add `public/<new-key>.txt` containing the new key as its only content.
3. Update `.github/workflows/indexnow.yml`'s `key` and `keyLocation` fields.
4. Keep the old `public/<old-key>.txt` around for a week as a grace period.
5. Commit + push. Next IndexNow submission uses the new key.

### Manual on-demand submission

```bash
curl -X POST 'https://api.indexnow.org/IndexNow' \
  -H 'Content-Type: application/json; charset=utf-8' \
  -d '{
    "host": "trybarakah.com",
    "key": "140db6db4ee38df9d39565683ff0b1734617c449d136e11d8e6fb0d7502663ab",
    "keyLocation": "https://trybarakah.com/140db6db4ee38df9d39565683ff0b1734617c449d136e11d8e6fb0d7502663ab.txt",
    "urlList": [
      "https://trybarakah.com/learn/new-article"
    ]
  }'
```

Response codes:
- **200** — success, queued for crawling
- **202** — accepted, verification pending (same outcome, different provider)
- **400** — bad request (usually malformed JSON or key-location mismatch)
- **403** — key file not accessible at `keyLocation`
- **422** — URL not on the `host` domain
- **429** — rate limited (wait + retry)

## 3. Google Search Console (one-time manual)

Google does not have an auto-submit API for general SEO pages. The
official Indexing API is restricted to `JobPosting` and
`BroadcastEvent` schemas only — using it for regular pages violates
Google's policy and can lead to deindexing.

The correct flow is a **one-time manual sitemap submission**, then
Googlebot crawls on its own schedule (typically within 24-72 hours
of any sitemap change).

### Steps

1. Go to https://search.google.com/search-console.
2. Select the `trybarakah.com` property (or add it if not yet verified; verification = DNS TXT record via your registrar, or an HTML `google-site-verification` meta tag in `src/app/layout.tsx`).
3. Left nav → **Sitemaps**.
4. Enter `sitemap.xml` in the "Add a new sitemap" field.
5. Click **Submit**.
6. Wait 5-10 minutes; the status should move to **Success** with URL count = ~55.

### After submission

- Googlebot re-fetches the sitemap at its own cadence (roughly daily for active domains).
- To force an immediate recrawl of a specific page:
  - Left nav → **URL Inspection**.
  - Paste the URL.
  - Click **Request Indexing**.
  - 10-20 recrawl requests per day are allowed.

### If the property is not yet verified

Add a verification meta tag to `src/app/layout.tsx`:

```tsx
<meta name="google-site-verification" content="<token-from-search-console>" />
```

Or add a DNS TXT record `google-site-verification=<token>` via your registrar. DNS verification is preferred because it works at the domain level and allows adding subdomains later (e.g. `api.trybarakah.com`) without re-verifying.

## 4. Bing Webmaster Tools (one-time manual, optional)

IndexNow already covers Bing's crawl trigger, but for the full Bing
dashboard (index coverage reports, backlink data, Core Web Vitals for
Bing search results):

1. Go to https://www.bing.com/webmasters.
2. Import the site from Google Search Console (one click after GSC is set up).
3. Submit `sitemap.xml` in the Sitemaps section — same flow as GSC.

## 5. What NOT to do

- ❌ **Don't** use Google's Indexing API for general pages. It's for `JobPosting` / `BroadcastEvent` only. Abuse → deindexing.
- ❌ **Don't** ping the old `google.com/ping?sitemap=...` endpoint — deprecated June 2023, silently discarded.
- ❌ **Don't** put unlisted / staging / dashboard URLs in the sitemap.
- ❌ **Don't** pay an "SEO submission service" — all legitimate paths are free.

## 6. Monitoring

- **Google**: Search Console → Coverage report shows indexed vs excluded URLs. Check weekly.
- **Bing**: Webmaster Tools → Site Explorer. Same cadence.
- **IndexNow submission history**: GitHub Actions tab → "IndexNow" workflow runs. Each run's Job Summary shows response codes from all three providers.
